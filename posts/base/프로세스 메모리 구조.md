# 프로세스 메모리 구조

## 코드
> 실행할 프로그램의 코드 저장 (함수, 메소드, 제어문, 상수)
>> CPU는 이 영역에서 명령어를 하나씩 가져와 처리
>>> 컴파일 시 결정
>>>> Read-Only 속성

## 데이터
> 전역변수, 정적변수(Static)
>> 프로그램 종료 시 소멸

## 힙
> 동적할당, 인스턴스
>> `메모리 주소 값`에 의해서만 참조되고 사용되는 영역
>>> 런타임 시 크기가 결정

## 스택
> `지역변수, 매개변수, 리턴값` 등 잠시 사용되었다가 사라지는 데이터를 저장
>> `함수 호출 시` 할당되고 `함수 반환 시` 소멸
>>> 컴파일 시 크기가 결정

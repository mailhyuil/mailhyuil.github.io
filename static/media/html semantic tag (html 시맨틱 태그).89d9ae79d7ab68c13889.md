# HTML Semantic Tag

> 의미가 있는 태그를 사용

> > 채워질 데이터를 기반으로 데이터를 가장 잘 설명하고 나타낼 수 있는 태그를 활용한다.

> > > p 태그를 남용하지 않기

- 장점
  - 검색엔진최적화(SEO)
  - 시각장애가 있는 사용자가 스크린 리더를 사용하여 페이지를 탐색할 때 도움
  - 의미가 있기에 div 태그보다 찾기가 쉽다
  - 시맨틱 웹을 사용하면 애플리케이션, 기업 및 커뮤니티에서 데이터를 공유하고 재사용할 수 있다.

![](/img/semantic_tag.png)

1. header : 페이지의 제목과 같은 소개 내용
2. nav : 메뉴, 목차
3. aside : 사이드바 또는 콜아웃 상자
4. main : 지배적인 콘텐츠 영역
5. section : 그체적인 시맨틱 태그가 없는 문서의 독립적인 영역
6. article : 그 자체로 의미가 있는 웹사이트의 부분, 독립적으로 배포 또는 재사용되도록 의도된 문서
7. footer : 작성자 정보, 저작권 데이터 또는 관련 문서 링크
8. em, strong : 강조

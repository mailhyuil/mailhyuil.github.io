# git branch 용어

## feature-branch & topic-branch

> 둘 다 같은 용어
>
> > 특정한 기능을 개발하기 위한 브랜치
> >
> > > feature branch에서 개발하고 완료되면 develop 브랜치에 merge한다. (squash)

## develop-branch

> main 브랜치에 병합되기 전 단계
>
> > develop branch에서 개발하고 완료되면 main 브랜치에 merge한다. (rebase)

# hotfix-branch

> 배포한 버전에 긴급하게 수정을 해야 할 필요가 있을 때 사용하는 브랜치
>
> > hotfix-라는 이름을 사용하는 것이 일반적이다.

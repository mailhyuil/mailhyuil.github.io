```sh
TCP 세션이 수립된 상태는 내부적으로 소켓(Socket)으로 표현됩니다. 소켓은 TCP/IP 네트워크 프로그래밍에서 사용되는 개념으로, 양쪽 연결된 호스트 간의 통신을 담당하는 역할을 합니다.

클라이언트와 서버 각각은 소켓을 생성하여 TCP 연결을 관리합니다. 소켓은 IP 주소와 포트 번호로 식별됩니다. 클라이언트 소켓은 클라이언트 IP 주소와 클라이언트가 사용하는 임시 포트 번호로, 서버 소켓은 서버 IP 주소와 서버가 사용하는 포트 번호로 구별됩니다.

TCP 세션이 수립되면, 클라이언트 소켓과 서버 소켓은 각각 로컬 주소와 원격 주소를 가지게 됩니다. 로컬 주소는 소켓이 속한 호스트(로컬 컴퓨터)의 IP 주소와 포트 번호를 나타내며, 원격 주소는 연결된 상대방(클라이언트 또는 서버)의 IP 주소와 포트 번호를 나타냅니다. 이를 통해 소켓은 자신이 어떤 연결에 속하고, 연결된 상대방과 어떤 주소로 통신해야 하는지를 알 수 있습니다.

소켓은 연결 상태를 유지하며, 데이터를 송수신하고 세션의 종료를 관리합니다. 데이터를 주고받을 때는 소켓을 통해 패킷을 생성하고 전송하며, 수신된 패킷을 소켓을 통해 응용 프로그램으로 전달합니다. TCP 세션이 종료되면 소켓도 닫히게 되어 해당 연결과 관련된 리소스를 해제합니다.

요약하면, TCP 세션은 클라이언트와 서버 각각의 소켓으로 표현되며, 소켓은 연결 상태를 유지하고 데이터 전송을 처리합니다. 소켓은 로컬 주소와 원격 주소를 기반으로 상대방과의 통신을 관리합니다.
```

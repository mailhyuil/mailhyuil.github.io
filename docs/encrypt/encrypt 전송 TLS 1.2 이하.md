# TLS 1.2 이하

```txt
1. 클라이언트 헬로: 클라이언트가 지원하는 TLS 버전, 암호 제품군, 무작위 값 전송
2. 서버 헬로: 서버가 선택한 암호 제품군, SSL 인증서, 무작위 값 전송
3. 인증: 클라이언트가 서버의 인증서를 검증
4. 예비 마스터 암호 전송: 클라이언트가 서버의 공개 키로 암호화된 예비 마스터 암호 전송
5. 서버가 해독: 서버의 개인 키로 예비 마스터 암호 해독
6. 세션 키 생성: 클라이언트와 서버가 세션 키 생성
7. 서로 준비 완료 메시지 교환: 세션 키로 암호화된 "완료" 메시지 전송
8. 보안 연결 성립: 이후 통신은 세션 키를 이용한 대칭 암호화 사용
```

# algorithm base Big O 계산

**1억 번** 연산하는데 걸리는 시간 **1~5초**

## N = 20

- 가능: O(2^N), O(N·2^N), 백트래킹 전탐색
- 불가: 없음(웬만한 건 다 가능)

## N = 10^2 (100)

- 가능: O(N^3), O(N^2), O(N log N), O(N)
- 불가: O(2^N), O(N!)

## N = 10^3 (1,000)

- 가능: O(N^2), O(N log N), O(N)
- 불가: O(N^3) 이상(플로이드 정도만 가능)

## N = 10^4 (10,000)

- 가능: O(N log N), O(N)
- 경계: O(N^2) ≈ 10^8 (상황 봐서)
- 불가: O(N^3)

## N = 10^5 (100,000)

- 가능: O(N log N), O(N)
- 불가: O(N^2)

## N = 10^6 (1,000,000)

- 가능: O(N), O(N log N) (여유)
- 불가: O(N^2)

## N = 10^7 (10,000,000)

- 가능: O(N)
- 위험: O(N log N)
- 불가: O(N^2)

## N = 10^8 (100,000,000)

- 가능: O(N)만 가능 (딱 기준선)
- 불가: O(N log N), O(N^2)

## N = 10^9 (1,000,000,000)

- 가능: O(log N), O(1), O(√N)
- 불가: O(N)

## N = 2^31 ≈ 2.1×10^9 (2,147,483,648)

- 가능: O(log N), 분할정복, 이분 탐색
- 불가: O(N)

## N = 10^18 (1,000,000,000,000,000,000)

- 가능: O(log N), O(1) 수학, 분할정복
- 불가: O(√N), O(N)

# git Semantic Commit Messages

[Semantic Commit Messages](https://gist.github.com/joshbuchea/6f47e86d2510bce28f8e7f42ae84c716)

## `<type>(<scope>): <subject>`

> (e.g. feat:add some feature)
>
> > (e.g. chore:update some code)

## feat

> 새로운 기능이 추가됨

## fix

> 버그 fix

## docs

> documentation이 변경

## style

> formatting의 변경
>
> > production 코드는 변동이 없음

## refactor

> 리팩토링

## test

> test가 추가됨

## chore

> 자잘한 작업이 업데이트 될 때
>
> > production 코드는 변동이 없음

# 서비스 보안

```txt
서버의 포트는 80, 443 (nginx), 22 (sshd) 만 열려있다. (sshd는 필요할 때만 knockd로 열린다.)
nginx와 sshd의 설정에는 보안 취약점이 없다. (fail2ban으로 위험 IP 차단 또는 내 IP만 접속 가능)
비밀번호는 매우 어렵고 절대 유출되지 않는다 또한 주기적으로 업데이트 (비밀번호는 숫자 + 대소문자 + 특수문자 10자리 난수면 무차별 대입으로 1,900년이 걸림)
해커는 서버에 물리적으로 접근할 수 없다.
시크릿키는 절대 유출되지 않는다.
서버와 시스템에서 새로운 취약점이 발견되지 않는다. 또한 서버와 시스템을 꾸준히 업데이트
백엔드 서버와 클라이언트 앱 로직에는 어떠한 보안 취약점이 없다.

위의 가정하에 해커가 내 서비스를 해킹하는 건 거의 불가능하다.
```

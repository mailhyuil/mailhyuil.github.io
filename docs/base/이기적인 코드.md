# 이기적인 코드

```txt
코드는 이기적이여야 한다. 클래스나 함수가 자신에게 주어진 책임을 계속 남에게 떠넘기 코드가 결과적으로 클린 코드 만들어 낸다고 생각했다..
예를 들어서 피자를 주문하는 프로그램이 있다면 피자 스토어 클래스가 하는 일은 "클라이언트가 요청을 하면 피자를 제공하는 책임"만 갖고 나머지 일들(피자 만들기 등등..)의 책임은
다른 클래스 혹은 함수에게 떠넘긴다. 마치 클래스가 "내 책임은 딱 여기까지야 다른 문제가 생긴다면 그건 니 책임이야!!"라고 이기적으로 행동하는 것처럼 말이다..ㅋㅋ 그 책임을 떠맡은 피자를 만드는 코드는 다시 "자신은 무조건 피자를 만드는 일만" 하겠다며 나머지 책임(재료 손질, 재료 구하기..)를 다른 클래스 혹은 함수에게 떠넘긴다. 이러한 이기적인 행동이 객체 지향 원칙 중 하나인 단일 책임 원칙을 잘 따르는 코드를 구현하게 되고 느슨한 결합(loose coupling)을 구현하게 된다.

이렇게 책임을 계속 떠넘기게 되면 결과적으로 DI를 구현하게 되고 프레임워크의 구조가 만들어 진다. 우리가 보다 안정성있는 프로그램을 만들기 위해 프레임워크를 사용하는 이유가 바로 이렇게 이기적으로 코딩을 하기 위해서다.

팩토리 메소드 패턴을 사용하면 이러한 코드를 생성할 수 있다.
```

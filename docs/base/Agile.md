# Agile

전통적인 waterfall 방법론과 달리 점진적인 개발을 위한 방법론  
빠른 출시와 피드백을 통한 반복적인 개발을 위한 방법론

## Scrum

- 짧은 반복 주기(스프린트)와 정기적인 미팅(데일리 스크럼)을 통해 팀의 투명성과 적응력을 높입니다.
- 제품 백로그(Product Backlog)와 스프린트 백로그(Sprint Backlog)를 사용해 작업의 우선순위를 관리
- 스프린트 리뷰와 회고를 통해 지속적으로 성과를 평가하고 개선점을 도출

## Kanban

- To Do, In Progress, Done 3개의 상태를 사용해 작업의 진행 상태를 관리
- 작업 흐름 시각화

## XP (eXtreme Programming)

- 테스트 주도 개발(TDD)
- 페어 프로그래밍(Pair Programming)
- 지속적 통합(CI)

## Lean

- 가치(Value)는 극대화하고, 낭비(Waste)는 철저히 제거하여, 고객에게 가장 빠르게 전달하는 것
- 가장 적은 자원으로 가장 빠른 학습을 실현하는 프로세스
- 중복된 작업을 제거하고, 비효율적인 작업을 제거하여 효율적인 개발을 실현

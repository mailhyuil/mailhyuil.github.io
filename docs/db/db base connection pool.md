# db base connection pool

> 클라이언트와의 연결을 미리 생성해두고, 요청이 들어올 때마다 미리 생성해둔 연결을 사용하는 방식
>
> > 너무 많으면 메모리에 연결 정보를 저장해두는데 비용이 많이 들 수 있음
> >
> > 커넥션을 관리하는데 생기는 cpu 오버헤드
> >
> > > 적으면 연결을 생성하는데 시간이 걸릴 수 있음

## 일반적인 커넥션 풀 설정 가이드

```txt
서버 스펙	        추천 max_connections 값
2코어/4GB RAM	   10~30
4코어/8GB RAM	   30~50
8코어/16GB RAM     50~100
16코어이상/100이상   (테스트 필요)
```

# db index B+ Tree vs Hash vs Bitmap

## B+ Tree

> 정렬된 데이터에 빠르게 접근할 수 있는 자료구조
>
> > 쇼핑몰 상품 목록, 게시판 글 목록, 유저 검색 등 범위 / 정렬 / LIKE 검색이 필요한 경우 사용

## Hash

> 일치하는 값에 빠르게 접근할 수 있는 자료구조
>
> 값이 정렬되어 있지 않기 때문에 범위 조회가 불가능하다.
>
> > 로그인, API 키 인증, 캐시 룩업 등 “값이 완전히 일치해야만 찾을 때” 사용

## Bitmap

> 열(column) 의 각 가능한 값을 비트(bit) 로 표현해서 0/1 패턴으로 검색을 빠르게 수행하는 방식.
>
> > 통계 / 리포트 / OLAP 환경 “남녀, 지역, 등급, 상태값” 처럼 중복 많은 컬럼 분석 시 사용
> >
> > `COUNT`, `AND/OR/NOT`, `GROUP BY`, `DISTINCT`, `IN`, `=`, `BETWEEN`, `HAVING COUNT > n` “비트 연산으로 처리 가능한 집계”에만 빠르다.

# ad Cookie & Sandbox

## 서드파티 쿠키 (3rd Party Cookie) 방식

1. 유입: 유저가 광고를 클릭해 A 사이트에 들어옴.
2. 식별자 생성: 광고 스크립트가 브라우저에 고유 ID(쿠키)를 저장함.
3. 추적: 유저가 B 사이트, C 사이트를 가도 동일한 쿠키 ID를 서버로 전송해 유저의 모든 행적을 기록함.
4. 전환: 나중에 구매가 일어나면 쿠키 ID를 대조해 "이 사람이 3일 전 광고 본 그 사람"임을 100% 확신함.

## Topics API 방식 (관심사 기반 타겟팅)

1. 관찰: 브라우저(크롬)가 유저의 최근 방문 기록을 스스로 분석함.
2. 그룹화: 유저를 식별하는 대신 '운동화', '여행', '요리' 같은 **관심사 키워드(Topics)**를 유저에게 부여함.
3. 제공: 광고주가 "이 유저 누군지 알려줘"라고 하면, 브라우저는 "이름은 몰라도 이 사람은 '요리'에 관심 있어"라고 주제만 전달함.
4. 결과: 개인을 식별하지 않고도 관심사에 맞는 광고를 노출함.

## Attribution Reporting API 방식 (성과 측정 기반)

1. 클릭 기록: 유저가 광고를 클릭하면 브라우저가 클릭 정보를 로컬에 저장함.
2. 구매 발생: 나중에 구매가 일어나면 브라우저가 "아까 저장한 클릭 정보랑 매칭되네"라고 판단함.
3. 지연 전송: 브라우저는 유저가 누구인지 알 수 없게 데이터를 암호화하고, 무작위 시간(24~48시간 뒤)에 서버로 전송함.
4. 집계 보고: 백엔드 서버는 개별 유저 데이터가 아니라 "오늘 이 캠페인에서 총 10건의 구매가 있었다"는 **집계된 결과(Summary Report)**만 받음.

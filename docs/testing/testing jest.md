# jest

> .spec.ts/.test.ts 파일 또는 \_\_tests\_\_ 디렉터리 안에 있는 파일들은 모두 테스트 파일로 인식
>
> > describe로 test suite를 만들고 내부에서 it(test)로 테스트 케이스를 만들 수 있다.
> >
> > > it과 test는 동일한 함수이다.

## install

```sh
npm i -D jest
npm i -D @types/jest
npm i -D ts-jest
npm init jest
```

## 명령어

```sh
jest
jest --watch
jest --watchAll
```

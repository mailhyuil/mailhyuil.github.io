# 서비스 보안

```txt
포트가 80, 443 (nginx) 22 (sshd) 만 열려있음
nginx와 sshd에는 어떠한 보안적으로 위험한 설정이 없음 (ssh는 필요할 때만 knockd로 열리고 fail2ban으로 차단 내 ip만 접속 가능)
비밀번호는 매우 어렵고 절대 유출되지 않고 주기적으로 업데이트 됨 (비밀번호는 숫자 + 대소문자 + 특수문자 10자리 난수면 무차별 대입으로 1,900년이 걸림)
해커는 서버에 물리적으로 접근할 수 없음
pem키도 절대 유출되지 않음
서버와 시스템을 꾸준히 업데이트 됨
백엔드 서버와 클라이언트 앱에 어떠한 보안적으로 문제되는게 없음
내 시스템에서 새로운 취약점이 발견되지 않음 (발견되도 즉각 업데이트 함)

위의 가정하에 해커가 내 서비스를 해킹하는 건 거의 불가능함
```

# db index

> 정렬된 데이터를 가진 '테이블'
>
> > 이를 통해 탐색범위를 최소화
> >
> > > 가장 나이가 어린 사람을 db에서 찾고 싶을 때 정렬이 되어 있다면 맨 앞의 데이터만 보면 됨 O(1)
> > >
> > > > 인덱스의 핵심은 검색 범위를 최소화 하는 것

## B+ tree

> 인덱스 자료구조로 많이 사용된다.
>
> > tree가 균형을 이루고 높이가 높지 않기 때문

## 클러스터 인덱스 vs 넌클러스터 인덱스

### 클러스터 인덱스

> 페이지를 알기 때문에 바로 그 페이지를 펴는 것

### 넌클러스터 인덱스

> 목차에서 찾고자 하는 내용의 페이지를 찾고 그 페이지로 이동하는 것.

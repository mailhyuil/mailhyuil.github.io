# db transaction lock

## optimistic lock (낙관적 락)

> 트랜잭션은 충돌하지 않는다는 가정
>
> > 트랜잭션 상황에서 시작 시 데이터 버전과 끝나는 시점의 데이터 버전이 다른 경우 실패처리를 하기 위해 사용한다. (postgres의 MVCC)
> >
> > > 성능이 좋다. (읽기가 많이 이루어지는 경우에 사용)
> > >
> > > 트랜잭션 충돌이 자주 발생하지 않을 것으로 예상될 때 사용

## pessimistic lock (비관적 락)

> 트랜잭션은 무조건 충돌한다는 가정
>
> > 실제로 다른 트랜잭션이 접근하지 못하도록 락을 걸어놓는다.
> >
> > > 공유락, 배타락, 읽기락, 쓰기락 등이 있다.
> > >
> > > > 데드락이 일어날 가능성이 있고 성능 저하가 있다. (읽기가 많이 이루어지는 경우에 좋지 않다.)
> > > >
> > > > 트랜잭션이 충돌이 자주 발생할 것으로 예상될 때 사용

## deadlock (데드락)

> 두 개 이상의 트랜잭션이 서로 상대방의 락을 기다리고 있어 무한 대기 상태에 빠지는 현상

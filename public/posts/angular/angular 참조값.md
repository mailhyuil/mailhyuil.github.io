# angular 참조값

> angular를 메모리 내에 저장된 "값"이 바뀌어야 변경을 감지
>
> > 윈시값은 그 자체로 값이기 때문에 변경을 쉽게 감지할 수 있음
> >
> > > 하지만 객체, 배열은 내부의 값이 바뀌어버리면 참조값은 그대로기 때문에 변경을 감지할 수 없음
> > >
> > > > 따라서 반드시 새로운 객체, 배열을 만들어서 새로운 참조값을 만들어야함
> > > >
> > > > > 리액트, flutter는 setState()를 사용해서 새로운 객체를 만들어서 새로운 참조값을 만들어서 변경을 감지함
> > > > >
> > > > > > vue의 경우는 프록시 패턴으로 참조값이 안바뀌어도 객체, 배열의 변경을 감지할 수 있음
